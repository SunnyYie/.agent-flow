# 上下文污染：长会话质量退化

> 90% 的"模型变差了"其实是上下文被污染，不是模型退步。

## 问题描述

在长会话中，LLM 的输出质量逐渐退化。常见误判是"模型今天不好用"，实际原因是上下文窗口中积累了大量无关/错误信息，稀释了有效指令的信号。

## 典型症状

1. **精度下降**: 回答越来越模糊，从精确方案变成泛泛而谈
2. **指令遗忘**: 忽略之前明确给出的规则和约束
3. **方向偏移**: 输出被对话中偶然出现的错误信息带偏
4. **重复错误**: 同一个错误反复出现，即使在后续对话中被纠正

## 根因分析

### 1. 信息稀释（最常见）

```
有效指令占比：
会话初期: ████████████████░░░░ 80%
会话中期: ████████░░░░░░░░░░░░ 40%
会话后期: ████░░░░░░░░░░░░░░░░ 20%
```

大量日志输出、错误信息、调试代码占据上下文，有效指令被淹没。

### 2. 错误信号传播

一次错误的推理/输出进入上下文后，后续生成会参考这个错误，形成恶性循环。

### 3. 方向冲突

对话中途改变需求方向，导致上下文中同时存在相互矛盾的指令。

## 解决方案

### 即时修复

| 方法 | 操作 | 适用场景 |
|------|------|---------|
| `/compact` | 精简上下文 | 轻度退化，保留会话 |
| 新开会话 | 完全重置 | 严重退化，无法恢复 |
| Agent 隔离 | 用独立 Agent | 子任务有大量调试输出 |

### 预防措施

1. **降低 auto-compact 阈值**:
   ```bash
   export CLAUDE_AUTOCOMPACT_PCT_OVERRIDE=50  # 默认约95%，改为50%
   ```

2. **减少上下文噪音**:
   - 不要让 Agent 读整个日志文件，先 grep 过滤
   - 调试输出用独立 Agent，不污染主对话
   - 大文件读取指定行范围

3. **任务拆分**:
   - 一个会话一个主任务
   - 调试/探索用子 Agent
   - 批量任务用 `/batch` 而非单会话

4. **定期清理**:
   - 上下文超过 60% 时主动 `/compact`
   - 完成一个子任务后压缩
   - 用 `--bare` 启动轻量会话处理简单任务

## 反模式

- ❌ 在一个会话中处理 5+ 个不相关任务
- ❌ 把完整错误日志贴到对话中
- ❌ 长时间不 `/compact`
- ❌ 用主对话做大量代码探索（应用 Explore Agent）
