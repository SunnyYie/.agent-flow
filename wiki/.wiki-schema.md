# Wiki配置规范

> 本文件定义 Wiki 知识库的统一结构规范，适用于全局 Wiki 和项目 Wiki。

## 目录结构规则

```
wiki/
├── purpose.md          # 项目方向与核心问题（项目Wiki独有）
├── INDEX.md            # 动态导航索引
├── GLOSSARY.md         # 术语枢纽（交叉链接中心）
├── TAG-INDEX.md        # 标签索引（tag → 文档映射，快速查找）
├── .wiki-schema.md     # Wiki配置规范（本文件）
├── topics/             # 主题枢纽（跨分类聚合视图）
│   └── {topic}.md      # 单个主题页（聚合patterns+pitfalls+concepts+tools）
├── patterns/           # 成功模式
│   └── {module}/       # 按模块组织
├── pitfalls/           # 踩坑记录
│   └── {module}/       # 按模块组织
├── concepts/           # 核心概念
├── decisions/          # 架构决策记录（ADR）
└── queries/            # 高价值Q&A存档
```

## 页面命名规则

1. **文件名**: kebab-case，如 `path-traversal-bypass.md`
2. **Wikilinks**: `[[filename|显示文本]]`，filename 不含 `.md` 扩展名
3. **分类路径**: 与 category + module 对应，如 pitfall+security → `pitfalls/security/`

## YAML Frontmatter 规范

```yaml
---
title: "页面标题"                          # 必填
category: pattern | pitfall | concept | decision | query | topic-hub  # 必填
module: workflow | security | environment | implementation | testing | architecture  # 必填
agents: [main, executor, verifier]          # 必填，哪些Agent应关注
scope: global | project                     # 必填，全局还是项目级
confidence: 0.0-1.0                         # 必填
sources: []                                 # 必填，来源（阶段任务/PR/issue）
created: YYYY-MM-DD                         # 必填
updated: YYYY-MM-DD                         # 必填
status: draft | verified | deprecated       # 必填
---
```

### 字段说明

| 字段 | 说明 |
|------|------|
| `agents` | 指定哪些Agent角色应关注此页面，用于定向检索 |
| `scope` | `global`=跨项目通用，`project`=项目特有 |
| `sources` | 经验来源（如阶段任务编号、PR号、issue号） |
| `status` | 知识生命周期状态 |

## 知识生命周期

| 状态 | 含义 | 转换条件 |
|------|------|----------|
| draft | 新创建，未经多阶段验证 | 默认初始状态 |
| verified | 经多个阶段验证有效 | confidence ≥ 0.9 且被 ≥ 2 个阶段引用 |
| deprecated | 已过时，被新模式替代 | 有更新的 wiki 页面替代，添加 `superseded_by` 字段 |

## Wiki Lint 规则

每 2-3 个阶段执行一次健康检查：

1. **断链检测**: wikilinks 指向的页面必须存在
2. **孤立检测**: 每个页面至少有一个入链或出链
3. **矛盾检测**: 同一主题不应有两个 status=verified 且结论冲突的页面
4. **过期检测**: status=deprecated 的页面仍被引用时发出警告

## 知识流动管道

```
项目 Wiki (.dev-workflow/wiki/)
    ↓ agent-flow reflect（手动触发，confidence ≥ 0.7, sources ≥ 2）
项目共享 Wiki (.agent-flow/wiki/)
    ↓ agent-flow wiki sync（手动触发，scope: global, abstraction: universal/framework）
全局 Wiki (~/.agent-flow/wiki/)
```

**晋升规则**：
- 晋升时只提取通用部分，不搬移项目特有内容
- 必须手动触发，避免项目特有知识污染全局
