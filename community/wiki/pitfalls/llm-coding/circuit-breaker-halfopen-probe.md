# 熔断器半开状态必须限制探测请求数

## 适用场景
任何实现熔断器（Circuit Breaker）模式的代码。

## 问题
定义了 `halfOpenProbeCount` 配置字段但在 `allowRequest()` 中未消费，半开状态下所有请求无条件通过，无法有效隔离尚未恢复的服务。

## 规则
1. 半开状态仅允许 `halfOpenProbeCount` 个探测请求通过
2. 探测成功 → `recordSuccess()` 恢复到 `closed`
3. 探测失败 → `recordFailure()` 回到 `open`
4. 超出探测数的请求 → 拒绝

## 检查清单
1. 接口中定义的字段是否全部在实现中被消费
2. 半开状态下的请求计数是否有限制
3. 半开状态下的成功/失败是否触发状态迁移
4. 熔断器单元测试中是否覆盖半开状态的全部三种路径（探测通过/探测失败/超出探测数拒绝）
