# 回滚/恢复函数不得虚假标记步骤完成

## 适用场景
任何多步骤回滚/恢复/降级操作的实现。

## 问题
回滚函数将所有步骤标记为 `completed`，但实际只自动执行了部分步骤，其余需要外部编排。运维人员依赖返回结果做决策，虚假完成可能导致遗漏关键步骤。

## 规则
1. 自动执行的步骤才标记为 `completed`
2. 需要外部执行的步骤标记为 `pending` 或 `requires_manual_action`
3. 函数注释明确说明哪些步骤自动执行、哪些需外部编排
4. 不追求"函数执行即完成"的简洁性

## 检查清单
1. 回滚函数的 `completedSteps` 是否只包含实际自动执行的步骤序号
2. 需外部编排的步骤是否标记为 `pending` 或在注释中明确说明
3. `status` 字段是否准确反映实际执行进度（部分完成 vs 全部完成）
4. 回滚结果的文档/注释是否区分"自动执行"和"需人工执行"的步骤
